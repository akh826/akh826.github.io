JRXML Parameter & PDF Preview

Upload a .jrxml file to detect parameters, enter values, and preview PDF. Use local compile (Java on your PC) or optional JasperReports Server.

Parameter detection runs in the browser. PDF preview needs Node.js 18+ for jasper-proxy-server.js, plus JDK 11+ and a one-time Maven build for local JRXML compile.

Preview mode

Compiles your uploaded JRXML with JasperReports (Java), then fills the report. Reports with SQL need optional JDBC below. Simple templates work with an empty datasource.

Username and password are optional (HTTP Basic Auth). Report must already be deployed on the server.

Prerequisites & local proxy setup

Prerequisites (local compile)

  • Node.js 18+ — nodejs.org (node --version)
  • JDK 11+ — Adoptium (java -version)
  • Apache Maven (one-time build) — maven.apache.org (mvn -version)

Build Java runner (once)

cd tools/jrxml-report-preview
build-runner.bat

Or: cd java && mvn package → creates java/target/jrxml-local-runner.jar

Download & run proxy

Download jasper-proxy-server.js

Deployed script URL:

https://akh826.github.io/tools/jrxml-report-preview/jasper-proxy-server.js

Start the proxy (default port 8790):

node jasper-proxy-server.js

From this repo:

node tools/jrxml-report-preview/jasper-proxy-server.js

Local mode: POST /api/local/preview with your JRXML + parameters.
Server mode: deploy report on Jasper Server; URI is the path after /rest_v2/reports/.
