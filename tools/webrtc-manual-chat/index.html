WebRTC Manual Signaling Chat

A serverless 1-to-1 chat demo. Two browsers exchange WebRTC session data by copy-paste (or any side channel). Once connected, messages travel directly over a WebRTC data channel.

This is a peer-to-peer demo for two people, not a multi-user chat room. You need a second device or browser profile. STUN is used only for NAT traversal; no chat messages pass through any server.

Choose a role to begin signaling.

Signaling

  1. Create an offer and send it to the guest.

  2. Paste the guest's answer below and apply it.

  1. Paste the host offer and create your answer.

  2. Send this answer back to the host.

How to try it

  1. Open this page on two browsers (or one normal window + one private window).
  2. On device A, stay on Host and click Create offer. Copy the offer text.
  3. On device B, switch to Guest, paste the offer, and click Create answer.
  4. Copy the answer back to device A and click Apply answer.
  5. When the badge shows Connected, both sides can chat. Use chat apps, email, or QR codes to move the offer/answer strings.
